# 项目宪法 v1.0

## 核心原则
1. **规范优先**：任何功能开发必须先完成 spec 文档。
2. **测试先行**：所有功能必须按 TDD（红-绿-重构）进行。
3. **三层测试覆盖**：单元测试（60%）+ Widget 测试（30%）+ 集成测试（10%）。

## 技术标准
- Flutter 3.16+ / Dart 3.0+
- 测试工具：flutter_test + integration_test
- 覆盖率要求：整体 ≥ 80%

## 测试纪律
- 每个公共函数必须有单元测试
- 每个可复用 Widget 必须有 Widget 测试
- 每个用户故事必须有集成测试
- 禁止硬编码，测试数据使用环境变量

## CI 门禁
- PR 合并前必须通过所有测试
- 覆盖率下降则合并失败
